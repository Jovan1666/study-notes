# 新增文章格式规范

> 每次让 AI 上传新内容时，把本文件粘贴给 AI，让它按照以下规范执行。

---

## 统一结构：目录页 + 子章节文件

每篇文章采用以下结构，**不得合并成单文件**：

```
your-topic/
├── README.md              ← 目录页（必须）
├── 00_导读.md             ← 导读/背景（可选）
├── 01_第一章标题.md
├── 02_第二章标题.md
├── 03_第三章标题.md
└── 04_第四章标题.md
```

---

## 目录页（README.md）必须包含

1. **标题**：文章标题
2. **摘要引用**：一句话概括
3. **这篇讲什么**：2-4 句话说明背景、适合谁、建议怎么读
4. **目录表格**：每章标题 + 一句话简介
5. **底部导航**：返回上级分类 + 返回首页

## 子文件命名规则

- `00_` 前缀用于导读/背景
- `01_` `02_` ... 用于正文章节
- 用**中文**命名，如 `01_基础概念.md`

---

## 内容要求

- **原文一个字都不能改**：只拆分，不修改内容
- 表格内容从原文提取，不要自己发挥
- 底部导航路径按实际目录结构填写

---

## 上传到对应分类

- AI 工具类 → `ai-tools/`
- 认知成长类 → `self-growth/`

放入前先在分类 README 的表格里加入新文章的条目（标题 + 简介 + 日期）。

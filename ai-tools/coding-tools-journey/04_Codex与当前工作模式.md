# 四、Codex 与当前工作模式

> Codex 桌面端深度使用、现在的 VSCode + 插件协同模式

---

## Codex 桌面端 —— 扔任务给它，不用管

我之前也用过 Codex，但那时候模型速度实在太慢，只有需要强推理的任务才会用。gpt 5.4 出来之后体验有不少提升，加上 Codex 的 Windows 桌面端上线，就开始深入使用了。

OpenAI 那段时间给出了不少优惠政策来吸引用户，Codex 的额度和 ChatGPT 网页版是独立的，不共享，这也是一个加分项。

文本处理能力比之前强了很多，codex 5.3 和 gpt 5.4 之间感觉不是一个档次。最让我喜欢的用法是：准备好测试代码，把任务扔给它，赋予权限之后它会一直工作到完成，中间不需要我确认任何步骤。当然实际上任务不会百分之百全部完成，但不用反复确认「要不要继续」这一点就已经很舒服了——这个体验和 Warp 当时给我的感觉很像。

OpenAI 模型的缺点是速度慢，但作为大模型领域的开山鼻祖，能力还是在的。最近 Codex 又有新更新，还没来得及体验。

---

## 现在的工作模式

目前回归了 VSCode，在里面配置了 Claude Code、Codex 和 OpenCode 的插件，主力用 CC。配合 cc-switch 来管理模型 API，可以灵活切换 opus 4.6 和各种国产模型，也省去了每次登录账号的麻烦。

主要就是 Claude Code 和 Codex 桌面端搭配使用。Gemini 用得不多，主要是觉得它降智太快，稍微复杂一点就开始搞不清楚我在说什么了。CC 配合 skill 写出来的界面效果我觉得不比 Gemini 差。

还有一点前面提到过：Kiro 和 Antigravity 后来都被我用来反代 opus 4.6 模型。当时这两个平台还能用，但现在风控越来越严，又卡又慢，基本不再走这条路了。

# 三、转向原生 Agent 工具

> Claude Code、OpenCode

---

## Claude Code —— 真正的 Agent

大约 25 年 11 月，在网上刷到 Claude Code 很厉害，就去闲鱼找人代充，花了 180 块买了个 Claude 会员。初次体验是命令行的工作方式，当时很不习惯，现在已经完全接受了。最开始只是在网页端用它写文档，其实并不懂该怎么充分发挥它的能力。

到 26 年初，OpenCode 开始爆火，我就把这两个工具放在一起对比学习，认真研究 agent 该怎么用。我的结论是：Claude Code 肯定比 OpenCode 强。当然这不是崇洋媚外，是基于实际使用体验说的。

CC 对国内用户其实很不友好，风控严重。但架不住 agent 能力确实强，模型也强，工作起来效率高。

印象最深的功能是 agent team：一个大脑 agent 负责统筹，创建子 agent 并分配任务，各做各擅长的部分，效率大幅提升。但 token 消耗是真的大，很多时候根本不知不觉就消耗了很多。还有 fast 模式，六倍 token 消耗，效率更高——我只开过一次，一句话花了六块钱，之后就不太敢用了。

CoWork 功能我还没用过，但看了不少技术博主的视频，感觉也很强。最近出的一些新产品，配置比之前简单很多，但 CC 的问题是每次执行操作都要来问我能不能继续，需要手动确认下一步，这一点用起来有点打断节奏。

具体的使用方式网上有很多教程，这里就不展开了。

---

## OpenCode —— 开源，对国内友好

OpenCode 开源，GitHub 上 star 数超过 40K，对国内用户做了优化，这是它相比 CC 的优势所在。

它有一个很厉害的插件叫 omo（oh-my-opencode），是一种多智能体协作的方式：用一个强力模型作大脑，然后把任务分配给各自擅长的模型去执行。思路和 CC 的 agent team 类似，但 OpenCode 对国内用户更友好，而且提供不少免费额度，最新的 mimo-v2-pro 模型也有免费额度，适合小白先上手体验。

不过它有一个持续存在的 bug：用久了会突然批量输出一堆符号，还停不下来。这个问题在我使用期间一直没解决。

# 二、深度使用：IDE 续与插件

> Cursor 深度使用、CodeBuddy、Antigravity、Augment、Cline，以及告别订阅制 IDE 的原因

---

## Cursor（深度使用）—— 编程 IDE 里的苹果

离开 Warp 之后又回到了 Cursor，这一次用得很深。第一次用 Cursor 只是打开了一扇门，这次算是真正住进去了。累计消耗了大约 7000 刀的额度，换算下来可能有几百亿到几千亿 tokens。

Cursor 是我觉得生态最完善的编程 IDE。对刚入门的人来说上手很快，四个模式的名字就把功能说清楚了：ask 是和 AI 交互聊需求，agent 是调用工具配合开发文档去开发，plan 是制定开发文档，debug 是查 bug、在可能出问题的地方插测试代码逐步定位。

它还支持同时开多个模型来回答同一个问题，让你自己挑最好的那个。但我用下来不太接受这种方式——AI 输出的信息量已经很大了，同时接收一堆相似的答案再去对比，感觉太累。我的做法是按任务类型指定模型：设计和前端用 Gemini，快速搭后端代码用 Claude，测试、改 bug、针对性修改用 GPT。这样既发挥了各个模型的强项，也只需要处理一方的输出。

MCP 和 skill 的配置是 Cursor 的强项，很多 MCP 一键配置，还有专门管理和创建 skill 的地方。在这方面它确实像手机里的苹果——生态成熟，用起来顺，初入这个领域时它就是我的白月光，没什么好喷的。

---

## CodeBuddy —— 简单用过，不多评价

腾讯出品，简单体验过，没有深入了解，所以不作详细评价。国产工具还是希望能越做越好。

---

## Antigravity —— 额度一降再降

初次使用体验还不错，用谷歌账号登录，IP 满足要求的话能免费用不少额度，Claude 的额度不少，谷歌自研的 Gemini 额度更多。

但问题也很明显：IP 限制严，很多想用的人直接被挡在门外；额度一降再降，即使是付费用户也很快就被限额。没有深入体验过这个工具本身。

同样，Antigravity 的故事也没结束——后来我通过它反代 opus 4.6 模型使用过一段时间，不过现在风控严重，又卡又慢，体验很差。

---

## Augment / Cline（已改名）—— 插件阶段的两个节点

Augment 体验过，网上评价挺好，但我用下来感觉任务执行太慢，消费也高，一点小问题就花了不少钱，用不起，没有深入使用。

Cline（现已改名）是一个转折点。它通过配置 API key 来调用大模型，这个玩法开启了我的下一个阶段——与其依赖订阅制的 IDE，不如直接走 API，自己控制模型和成本。

---

## 为什么离开那些 IDE

上面提到的这些编程 IDE，不算插件类的工具，如果不是公司统一配置或者资金比较充足，个人开发者现在其实已经不太适合用了。价格贵、额度少，国内各种薅羊毛的方式也相继被风控。于是我走向了下一个阶段。

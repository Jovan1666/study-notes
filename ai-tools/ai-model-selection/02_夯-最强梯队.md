# 二、夯——最强梯队

这三个模型就是目前我觉得最好的三个模型。

---

## Claude Opus 4.6 / Opus 4.5

Opus 4.6 毋庸置疑的最强模型，文本处理和编写能力，图片理解能力，代码编写能力各个都是顶级，没得说。

只可惜 Anthropic 对国内的风控实在严重，除了使用官方渠道充值会员，国内中转基本都是从 Krio 或者 Antigravity 这样的平台反代出来，模型能力其实受到了限制，再加上不同的编程 IDE 的系统提示词的干扰，会有一点干扰，但是还是挡不住模型的能力很强大，即使有这些干扰依旧是领先其他模型。

不过可惜的是国内，至少现在除了走官方渠道这个会有封号风险，如果走中转站渠道，基本都不稳定，经常没输出，或者被限速限流，而且中转站的水很深，不少商家会用模型映射或者在一个大号池里面掺杂一些 Opus 4.6 和其他模型，这样子糊弄用户。

Opus 4.5 就是弱一点的 Opus 4.6，但是依旧是其他模型很难达到的高度，模型能力依旧很强。

---

## GPT-5.4

GPT-5.4 可以说是在 Claude 国内严重风控时期的最佳选择，但是现在也开始了封号浪潮，不过力度并没有 Claude 这么大，至少我自己搭建了一个号池，然后自己使用 GPT-5.4 模型。

其实 GPT 也就是 Codex 5.1 / 5.2 / 5.3 这些模型，能力都没的说，推理能力很强，问题就是实在是慢，用 GPT-5.4 之前的模型和同期的 Opus 模型做个对比，Opus 模型可能 2 分钟跑完的问题，Codex 模型要跑 10 分钟，而且 OpenAI 可能算力也没那么充沛吧，我当时用 Codex 5.3 感觉这个模型写文本的能力实在是不行，就是即使分析到位了，还是惜字如金写的内容很少，就导致写的东西没什么看点。

但是 GPT-5.4 更新之后就发生了变化，GPT-5.4 等于是把 Codex 5.4 已经加进去了，然后基于自己 GPT 本身的语言处理能力，这是我自己的理解，这样加强了模型的文本能力，至少我现在使用 GPT-5.4 他能写出至少不让我觉得不行的项目文档，然后运行速度真的有非常大的提升，之前最大的问题得到了非常好的解决，加之 Claude 的风控浪潮，让 GPT-5.4 使用量上升。

Codex 的模型本来推理能力就强，所以我很喜欢用他来做测试，或者定位问题，后端代码能力也没得说，但是写前端确实审美不行。
